# ADR-0011: Gateway Mode `local` and Bind `lan` Required for Containers

**Status:** Accepted
**Date:** 2026-03-03
**Deciders:** Brent Bryson

## Context

During Phase 0 container testing (P0-3), the OpenClaw gateway refused to start with the error:

```
Gateway start blocked: set gateway.mode=local (current: unset) or pass --allow-unconfigured.
```

OpenClaw v2026.3.2 requires `gateway.mode` to be explicitly set to `"local"` for self-hosted deployments. Without it, the gateway exits immediately.

Additionally, the default `gateway.bind` value is `"loopback"` (127.0.0.1), which only listens on the container's loopback interface. With Docker bridge networking (`-p 18789:18789`), inbound traffic arrives on `eth0`, making the gateway unreachable from outside the container. Setting `bind: "lan"` makes the gateway listen on `0.0.0.0`.

## Decision

Every Tiger Claw container's `openclaw.json` must include:

```json
{
  "gateway": {
    "mode": "local",
    "bind": "lan"
  }
}
```

These fields are generated by `entrypoint.sh` and are not configurable per-tenant — they are infrastructure constants for containerized deployments.

## Consequences

**Positive:**
- Gateway starts reliably in Docker containers
- Port mapping (`-p`) works correctly with bridge networking
- `/readyz` health checks from the host succeed

**Negative:**
- `bind: "lan"` exposes the gateway on all container interfaces. This is acceptable because containers are isolated and port exposure is controlled by Docker `-p` flags. OpenClaw logs a warning: "Gateway is binding to a non-loopback address. Ensure authentication is configured before exposing to public networks." Token auth (`gateway.auth.mode: "token"`) satisfies this requirement.

## Source

- https://docs.openclaw.ai/gateway/configuration-reference
  - `mode`: "Gateway refuses to start unless `local`"
  - `bind`: "Docker note: the default `loopback` bind listens on `127.0.0.1` inside the container... Use `bind: "lan"` to listen on all interfaces."

## Status of Implementation

Complete. Implemented in `docker/customer/entrypoint.sh` as part of P0-3/P0-5b.
